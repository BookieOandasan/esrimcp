# Data Model: AI Chat Component with ESRI MCP

**Feature**: 003-map-ai-chat  
**Date**: 2026-06-03

---

## Entities

### ChatMessageSender

Union type ‚Äî identifies who produced a message.

| Value | Description |
|-------|-------------|
| `'user'` | A message typed and submitted by the human user |
| `'bot'` | A response generated by the system (success, error, or canned reply) |

---

### ChatMessageStatus

Union type ‚Äî lifecycle state of a message.

| Value | Description |
|-------|-------------|
| `'pending'` | Bot reply is loading (typing indicator shown) |
| `'success'` | Message delivered successfully |
| `'error'` | Message represents an error response |

---

### ChatMessage

A single entry in the conversation history.

| Field | Type | Nullable | Description |
|-------|------|----------|-------------|
| `id` | `string` | No | Unique ID (e.g., `crypto.randomUUID()`) ‚Äî used as DOM key |
| `sender` | `ChatMessageSender` | No | `'user'` or `'bot'` |
| `text` | `string` | No | Display text of the message |
| `timestamp` | `Date` | No | When the message was added to the session |
| `status` | `ChatMessageStatus` | No | `'pending'` \| `'success'` \| `'error'` |

**Validation rules**:
- `text` must be non-empty and ‚â§ 500 characters before sending
- `id` must be globally unique within the session
- A `pending` bot message is always replaced (not kept alongside) the final `success`/`error` message

---

### ChatSession

Holds the full state of the chat feature. Managed by `ChatService` as a singleton.

| Field | Type | Description |
|-------|------|-------------|
| `messages` | `Signal<ChatMessage[]>` | Ordered list of all messages, oldest first |
| `isProcessing` | `Signal<boolean>` | True while a query is in-flight |

**Lifecycle**:
- Initialised with a single welcome bot message on first access
- `messages` is never emptied mid-session except by explicit "Clear chat" action
- On clear: messages reset to `[welcomeMessage]`

---

### LocationQueryResult *(alias ‚Äî mapped from existing `GeocodeResult`)*

The chat feature does not define a new entity for geocoding results ‚Äî it reuses the existing `GeocodeResult` from `src/app/core/models/app.types.ts`.

| Field | Source | Description |
|-------|--------|-------------|
| `displayName` | `GeocodeResult.displayName` | Used as the bot reply text ("Found: {displayName}") |
| `longitude` | `GeocodeResult.longitude` | Passed to `MapService.centerOn()` |
| `latitude` | `GeocodeResult.latitude` | Passed to `MapService.centerOn()` |
| `score` | `GeocodeResult.score` | Used to select top result |
| `extent` | `GeocodeResult.extent` | Passed to `MapService.centerOn()` for zoom |

## State Transitions

```
[App starts]
  ‚Üí ChatService initialises messages = [welcomeMessage]

[User submits text]
  ‚Üí ChatService adds ChatMessage { sender:'user', status:'success' }
  ‚Üí ChatService adds ChatMessage { sender:'bot', status:'pending' }  ‚Üê typing indicator
  ‚Üí ChatService.isProcessing.set(true)

  [Geocoding succeeds]
    ‚Üí pending bot message replaced with { status:'success', text:'Found: {displayName}' }
    ‚Üí MapService.centerOn(result)
    ‚Üí isProcessing.set(false)

  [Geocoding fails with "No results"]
    ‚Üí pending message replaced with { status:'success', text:'I couldn\'t find that location...' }
    ‚Üí isProcessing.set(false)

  [Geocoding fails with service error]
    ‚Üí pending message replaced with { status:'error', text:'The map service is temporarily unavailable...' }
    ‚Üí isProcessing.set(false)

[User clears chat]
  ‚Üí messages reset to [welcomeMessage]

[Offcanvas closes / reopens]
  ‚Üí ChatService state unchanged (singleton survives component destroy)
```
